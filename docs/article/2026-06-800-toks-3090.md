# 808.7 decode tok/s from one RTX 3090 Ti

808.7 aggregate decode tok/s from a single RTX 3090 Ti. Eight vLLM streams, Qwen/Qwen3-30B-A3B-GPTQ-Int4, 4.40× scaling, ~101 tok/s per stream. Coherent output. Parsed tool calls: 35/36.

Not an autonomous multi-turn worker. A very fast single-shot fan-out engine.

Getting there took four dead ends — and the fastest dead end was generating pure gibberish.

**The goal**

Parallel local agent workers on one 24 GB card. Single-stream was solved early: byteshape's Qwen3.6-35B-A3B under llama.cpp Vulkan runs 127.7 tok/s with 100% toolcall and 5/5 agentic. One excellent worker. We wanted eight.

**Dead ends at a glance**

| Path | Good sign | Failure |
|---|---:|---|
| llama.cpp slots | simple concurrency | 1.25–1.27× at 4 streams |
| MTP/spec decode | +28% solo | −34% at 4 streams |
| vLLM 35B AWQ | 360.3 tok/s at 4 streams | incoherent output |
| compressed-tensors INT4 MoE | small enough | startup quant assertion |

**Dead end 1: a pool of slots is not a pool**

llama.cpp has slot parallelism. Four slots should land somewhere near 4× aggregate. Measured: 1.25× on Vulkan, 1.27× on CUDA. Four streams bought a quarter more tokens.

**Dead end 2: speculative decoding fights batching**

MTP/speculative decoding looked great solo: +28% single-stream decode on Gemma 12B. At four streams on CUDA it inverts — 114.6 tok/s aggregate with spec, 174.8 without. −34% for leaving it on. Speculative decoding spends the compute headroom that batching needs. Turn it off past one stream.

**Dead end 3: 360 tok/s of nothing**

vLLM scaled. C15, `mattbucci/Qwen3.6-35B-A3B-AWQ`, did 2.76× at four streams, 360.3 tok/s aggregate. Then the coherence check: temperature 0, "say hello". The reply was multi-script token salad. Every prompt, gibberish.

We eliminated the tool parser, the chat template, and mrope/vision-rope config loss one by one. The same weights score 1.000 toolcall under llama.cpp. Remaining suspect: that VL-architecture community AWQ quant under vLLM 0.22.1.

The lesson: throughput suites can't see corruption. 360 tok/s of garbage measures exactly like 360 tok/s of code. Gate first: temperature 0, trivial prompt, no tools. If it cannot say hello, the benchmark is invalid.

**Dead end 4: the quant format wall**

Next candidate, `cyankiwi/North-Mini-Code-1.0-AWQ-INT4`, died at startup: `Only symmetric quantization is supported for MoE`. Its compressed-tensors INT4 quant hits a MoE path vLLM 0.22.1 refuses. Classic AWQ/GPTQ repos avoid that assertion path entirely.

**The unlock, in four takes**

Qwen3-30B-A3B, official GPTQ-Int4. 15.77 GiB of weights, text-only architecture. Still took four takes, each fixing exactly one thing:

1. Quant size. 35B quants don't fit 24 GB with batch KV cache; this one leaves room for 8 streams.
2. Quant format. Symmetric GPTQ — dodges the MoE assertion above.
3. Tool parser. Gen-3 Qwen speaks hermes JSON, not `qwen3_xml`. With the wrong parser the model emitted perfect tool calls the parser threw away as plain text.
4. Probe budget. A thinking model can't say hello in 128 tokens — the probe burned its whole budget in the reasoning channel and returned blank.

**The number**

This is the long-decode ceiling, not the end-to-end agent number.

| Streams | Aggregate tok/s | Scaling | Per-stream tok/s |
|---:|---:|---:|---:|
| 1 | 183.8 | — | 183.8 |
| 2 | 309.8 | 1.69× | 155.4 |
| 4 | 534.4 | 2.91× | 133.9 |
| 8 | 808.7 | 4.40× | 101.3 |

Throughput used client-measured wall time from OpenAI-compatible usage counts. Loaded VRAM: 22,173 MiB. Tool calls: 35/36 strict. Coherent output, parsed tool calls, all verified.

The honest caveat: a 3-run agentic variance pass scored 3/5, 1/5, 2/5 — and the same two tasks failed every run. That's systematic, not noise. This config stays off autonomous multi-turn duty. Its lane is well-specified single-shot fan-out.

**Proof on real work**

32 real docstring items, 8 streams, 24.7 seconds, 32/32 AST-verified. On short-generation work the sustained number is 345 tok/s average with a 512 peak — prefill and verify gaps eat decode time. 808 is the long-decode ceiling. Full split in the companion piece: [3× faster, 74% cheaper](2026-06-local-subagent-fleet.md).

**Reproduce it**

vLLM 0.22.1:

```bash
vllm serve Qwen/Qwen3-30B-A3B-GPTQ-Int4 \
  --revision 9b534e4318b7ebc3c961a839f13eb18b1833f441 \
  --max-model-len 16384 \
  --max-num-seqs 8 \
  --gpu-memory-utilization 0.92 \
  --quantization gptq_marlin \
  --enable-auto-tool-choice \
  --tool-call-parser hermes \
  --reasoning-parser qwen3
```

Hardware: one RTX 3090 Ti, 24 GB. Harness, raw logs, model SHAs, and the full gibberish diagnosis: github.com/svankina/local_agents.
